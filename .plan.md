# 方案：用 Resend 替代 Supabase 发送所有认证邮件

## 背景

当前认证流程：前端 Supabase JS SDK 直接调用 `signUp()`，Supabase 自带邮件服务发送确认邮件。
由于 Supabase 自带邮件有限制（每小时 4 封），需要用 Resend 替代。

## 整体思路

**方案：Supabase Custom SMTP + Resend SMTP**

Resend 提供 SMTP 接口，Supabase 支持自定义 SMTP。只需要在 **Supabase Dashboard** 配置 Resend 的 SMTP 信息即可，**不需要改任何代码**。

这是最简洁的方案，因为：
- Supabase 仍然负责生成确认 token、管理用户状态
- 只是邮件投递通道从 Supabase 自带换成 Resend SMTP
- 前端/后端代码零改动

## 实施步骤

### 第 1 步：在 Resend 中验证发送域名

1. 登录 https://resend.com/domains
2. 添加你的域名（比如 `nutrimaster.bohrium.com` 或你拥有的其他域名）
3. 按照 Resend 指引，在 DNS 中添加以下记录：
   - **SPF** (TXT 记录)
   - **DKIM** (TXT 记录，3 条)
   - **DMARC** (TXT 记录，可选但推荐)
4. 等待 DNS 验证通过（通常几分钟到几小时）

> **如果暂时没有自己的域名**：可以先用 Resend 的 onboarding 域名 `onboarding@resend.dev` 测试，但只能发送到你自己注册 Resend 的邮箱。正式使用必须验证自己的域名。

### 第 2 步：获取 Resend SMTP 凭据

Resend 的 SMTP 信息是固定的：
- **Host**: `smtp.resend.com`
- **Port**: `465` (SSL)
- **Username**: `resend`
- **Password**: 你的 Resend API Key，即 `re_5QoDM5AK_4op4MWgMAkmiGN5RBNnQpPUg`

### 第 3 步：在 Supabase Dashboard 配置自定义 SMTP

1. 进入 Supabase Dashboard → 你的项目 → **Authentication** → **Email Templates** (或 **SMTP Settings**)
2. 找到 **Custom SMTP** 或 **Enable Custom SMTP** 选项，开启
3. 填入以下信息：
   - **Sender email**: `noreply@你的已验证域名`（如 `noreply@nutrimaster.bohrium.com`）
   - **Sender name**: `NutriMaster`（显示名称）
   - **Host**: `smtp.resend.com`
   - **Port**: `465`
   - **Username**: `resend`
   - **Password**: `re_5QoDM5AK_4op4MWgMAkmiGN5RBNnQpPUg`
   - **Minimum interval**: 可以设为 `0`（Resend 无频率限制）
4. 保存

### 第 4 步：（可选）自定义邮件模板

在 Supabase Dashboard → **Authentication** → **Email Templates** 中可以自定义：
- **Confirm signup**（注册确认）
- **Magic Link**（魔法链接登录）
- **Change Email Address**（更改邮箱）
- **Reset Password**（重置密码）

模板支持 HTML，可以加入 NutriMaster 品牌样式。

### 第 5 步：测试

1. 注册一个新账号，检查是否收到来自 Resend 的确认邮件
2. 检查发件人是否显示为你配置的地址
3. 确认邮件中的链接能正常跳转（`SITE_URL` 已在 `.env` 中配置为 `https://nutrimaster.bohrium.com`）

## 代码改动

**无需任何代码改动。** 所有配置都在 Supabase Dashboard 中完成。

## 注意事项

1. **API Key 安全**：Resend API Key 只填在 Supabase Dashboard 的 SMTP 密码栏中，不需要写进代码或 `.env`
2. **域名验证必须完成**：如果域名 DNS 验证未完成，Resend 会拒绝发送
3. **Supabase 邮件频率限制会被取消**：配置自定义 SMTP 后，Supabase 自带的每小时 4 封限制不再适用，改由 Resend 的配额控制（免费版每天 100 封，付费版更多）
4. **如果以后还想用 Resend 发送非认证邮件**（如通知、报告等），再在 `.env` 中添加 `RESEND_API_KEY` 并在代码中集成 `resend` Python SDK
